# Domain 01. 인증 및 회원관리

이 문서는 CheckMo의 핵심 진입점인 인증 시스템과 회원가입 단계별 구현 상세를 설명합니다.

## 1. 회원가입 프로세스 (Signup Flow)

회원가입은 `src/app/(public)/signup/[step]/page.tsx`를 중심으로 동적 라우팅을 통해 단계별로 진행됩니다. 각 단계의 상태는 `SignupContext`를 통해 공유됩니다.

### 가입 단계 (Steps)
1.  **약관 동의 (`terms`)**: 필수 약관 동의 절차.
2.  **이메일 인증 (`email`)**: `authService.verifyEmail`을 통해 인증 번호 발송 및 `confirmEmail`로 번호 확인. (소셜 가입 시 생략)
3.  **비밀번호 설정 (`password`)**: 계정 비밀번호 설정. (소셜 가입 시 생략)
4.  **프로필 설정 (`profile`)**: 닉네임 중복 확인(`checkNickname`) 및 기본 정보 입력.
5.  **프로필 이미지 (`profile-image`)**: 프로필 사진 업로드 (S3 Presigned URL 방식).
6.  **가입 완료 (`complete`)**: 가입 축하 및 메인 이동.

### 소셜 가입 가드 (Social Guard)
소셜 로그인 직후 추가 정보(닉네임 등)가 없는 사용자는 `isSocial=true` 파라미터와 함께 자동으로 `profile` 단계로 리다이렉트됩니다.

## 2. 이미지 업로드 로직 (S3 Presigned URL)

보안과 서버 부하 경감을 위해 클라이언트가 S3에 직접 이미지를 업로드하는 방식을 사용합니다.

1.  **URL 요청**: `authService.getPresignedUrl`을 호출하여 서버로부터 이미지를 업로드할 수 있는 임시 URL(`presignedUrl`)과 최종 이미지 경로(`imageUrl`)를 받아옵니다.
2.  **직접 업로드**: `authService.uploadToS3`를 호출하여 서버를 거치지 않고 S3 버킷으로 `PUT` 요청을 보냅니다.
3.  **경로 저장**: 업로드가 성공하면 서버에는 `imageUrl`만을 전달하여 회원 정보에 저장합니다.

## 3. 프로필 및 추가 정보 관리

사용자가 가입 후 또는 프로필 수정 시 사용하는 주요 로직입니다.

*   **닉네임 중복 확인**: `authService.checkNickname`을 통해 실시간으로 사용 가능 여부를 검증합니다.
*   **추가 정보 입력**: 직업, 관심사 등 사용자의 부가 정보를 `additionalInfo` API를 통해 저장합니다.
*   **세션 유지**: 로그인 성공 시 서버는 `accessToken`을 쿠키로 내려주며, 프론트엔드는 `credentials: "include"` 설정을 통해 이후의 모든 요청에 쿠키를 포함합니다.

## 4. 주요 파일 및 컴포넌트
*   `src/contexts/SignupContext.tsx`: 가입 단계별 데이터 보존.
*   `src/components/base-ui/Join/steps/`: 각 단계별 UI 컴포넌트 모음.
*   `src/services/authService.ts`: 인증 및 계정 관련 모든 API 호출.
