# Domain 02. 독서모임 (Club)

이 문서는 CheckMo의 핵심 기능인 독서모임 생성, 가입 및 관리 프로세스를 설명합니다.

## 1. 모임 생성 및 검색

사용자는 새로운 독서모임을 개설하거나 기존 모임을 검색하여 가입할 수 있습니다.

*   **모임 생성 (`createClub`)**: `CreateClubRequest` 타입을 사용하여 모임 이름, 소개, 카테고리 등을 서버에 전달합니다. 생성 전 `checkNameDuplicate`를 통해 모임 이름 중복 여부를 먼저 확인합니다.
*   **모임 검색 (`searchClubs`)**: 키워드, 카테고리, 필터 등을 기반으로 모임을 검색합니다. 커서 기반 페이지네이션(`cursorId`)이 적용되어 있어 무분별한 데이터 로드를 방지합니다.
*   **추천 모임**: 서버에서 제공하는 추천 알고리즘에 따라 `getRecommendations` API를 통해 메인 화면 등에 모임을 노출합니다.

## 2. 가입 및 멤버십 관리

사용자가 모임에 참여하기 위한 절차입니다.

1.  **가입 신청 (`joinClub`)**: 모임 ID와 신청 사유 등을 포함하여 가입을 요청합니다.
2.  **상태 확인 (`getMyStatus`)**: 사용자가 특정 모임에 대해 어떤 상태(미가입, 승인 대기, 멤버 등)인지 확인하여 UI를 분기합니다.
3.  **내 모임 목록 (`getMyClubs`)**: 현재 내가 가입한 모든 모임 목록을 가져옵니다.

## 3. 모임 홈 (Club Home)

가입한 멤버가 모임 진입 시 보게 되는 통합 대시보드입니다.

*   **통합 정보 (`getClubHome`)**: 모임의 기본 정보, 멤버 수, 공지사항 요약 등을 한 번에 불러옵니다.
*   **최신 공지 (`getLatestNotice`)**: 가장 최근에 등록된 공지사항을 상단에 노출합니다. 공지가 없는 경우 에러 처리 대신 `null`을 반환하도록 예외 처리가 되어 있습니다.
*   **다음 정기모임 (`getNextMeeting`)**: 예정된 다음 모임 일정을 확인합니다.

## 4. 모임 관리자 기능 (Admin)

모임을 개설한 방장(관리자)이 사용하는 기능입니다.

*   **정보 수정 (`getAdminClubDetail`, `updateAdminClub`)**: 모임 소개글, 이미지 등 상세 설정을 수정합니다.
*   **멤버 관리**: 가입 신청을 승인하거나 거절하는 등의 로직은 `clubMemberService`를 통해 분리되어 관리됩니다.

## 5. 주요 파일 및 서비스
*   `src/services/clubService.ts`: 모임 전반에 대한 API 호출.
*   `src/services/clubMemberService.ts`: 멤버 승인/탈퇴 등 멤버십 전용 로직.
*   `src/types/groups/`: 모임 관련 데이터 타입 및 API 요청/응답 스펙 정의.
