# Flow 01. 복잡한 회원가입 상태 관리

회원가입 플로우는 여러 페이지(단계)를 넘나들며 데이터를 누적시켜야 합니다.
이 문서는 `SignupContext`가 그 복잡성을 어떻게 해결하는지, 그리고 왜 Zustand가 아닌 Context API를 선택했는지를 설명합니다.

## 1. 왜 Context API인가? (Zustand와의 역할 분담)

| 기준 | `SignupContext` (Context API) | `useAuthStore` (Zustand) |
| :--- | :--- | :--- |
| **사용 범위** | 회원가입 화면 트리 내부에서만 사용 | 앱 전체에서 공유 |
| **생명주기** | 가입 완료 시 `resetSignup()`으로 즉시 초기화 | 앱이 살아있는 동안 유지 |
| **목적** | 다단계 폼의 임시 데이터 누적 | 영속적인 인증 상태 보존 |

> **결론**: 회원가입 중에만 필요한 임시 상태를 전역 Zustand에 담는 것은 책임 범위 초과(SRP 위반)입니다. `SignupContext`는 회원가입 레이아웃(`/signup/layout.tsx`) 하위에만 Provider가 존재하기 때문에, 다른 경로로 이동하면 자동으로 정리됩니다.

## 2. SignupContext 상태 구조 (`SignupState`)

`SignupContext`는 회원가입의 모든 단계 데이터를 하나의 상태 객체(`state`)에 담아 각 단계 컴포넌트가 읽고 쓸 수 있게 합니다.

```typescript
interface SignupState {
  agreements: Record<string, boolean>; // 1단계: 약관 동의 여부
  email: string;                        // 2단계: 이메일 주소
  verificationCode: string;             // 2단계: 인증 번호
  isVerified: boolean;                  // 2단계: 이메일 인증 완료 여부
  timeLeft: number | null;              // 2단계: 인증 번호 만료까지 남은 시간
  password: string;                     // 3단계: 비밀번호
  nickname: string;                     // 4단계: 닉네임
  isNicknameChecked: boolean;           // 4단계: 닉네임 중복확인 완료 여부
  profileImage: string | null;          // 5단계: 프로필 이미지 URL
  isSocial: boolean;                    // 소셜 로그인 경로 여부
  // ... (기타 intro, name, phone, selectedInterests 등)
}
```

## 3. 성능 최적화 패턴

`SignupContext`는 불필요한 리렌더링을 방지하기 위해 `useMemo`와 `useCallback`을 적극 활용합니다.

*   **`actions` 객체 메모이제이션**: `setEmail`, `setPassword` 등 모든 setter 함수를 `useMemo`로 감싸 상태가 변해도 함수 참조가 바뀌지 않게 합니다.
*   **`contextValue` 메모이제이션**: 최종적으로 Provider에 전달되는 `contextValue` 역시 `useMemo`로 묶어 불필요한 자식 컴포넌트 재렌더를 차단합니다.

## 4. 인증 타이머 관리 (`useRef`)

이메일 인증 번호 발송 후의 카운트다운 타이머는 `toastTimers`라는 `useRef`를 통해 관리합니다.

*   **왜 ref인가?**: 타이머 ID는 UI에 영향을 주지 않습니다. `state`에 담으면 타이머 설정 시마다 불필요한 리렌더링이 발생합니다. `useRef`에 담으면 리렌더링 없이 최신 타이머 ID를 안전하게 보관합니다.
*   **중복 방지**: `showToast` 호출 시 기존 타이머를 먼저 전부 `clearTimeout`하여, 이전 타이머가 겹쳐 오동작하는 버그를 방지합니다.

## 5. 데이터 흐름 (단계 전환 시)

```
[약관 동의 (terms)]
  ↓ onNext() 호출 → navigateTo('email') 또는 navigateTo('profile') (소셜인 경우)
[이메일 인증 (email)]
  ↓ setEmail(email), setIsVerified(true)를 SignupContext에 저장
[비밀번호 설정 (password)]
  ↓ setPassword(password)를 SignupContext에 저장
[프로필 설정 (profile)]
  ↓ setNickname(nickname) 등을 SignupContext에 저장
  ↓ 최종 API 호출 시 authService.signup({ email, password, nickname, ... }) 전송
[완료 (complete)]
  ↓ resetSignup() 호출 → SignupContext 초기화
```
