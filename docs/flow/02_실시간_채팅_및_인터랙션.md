# Flow 02. 실시간 채팅 및 인터랙션

이 문서는 `useMeetingRealtime` 훅을 중심으로, WebSocket(STOMP) 연결의 수립부터 메시지 수신 및 캐시 동기화까지 전체 흐름을 설명합니다.

## 1. 전체 아키텍처 요약

실시간 기능은 **3개의 레이어**로 관심사가 분리(SoC)되어 있습니다.

```
[컴포넌트]
    ↓ useMeetingRealtime(params) 호출
[useMeetingRealtime 훅] ← 연결 관리, 구독, 발행 로직
    ↓ createMeetingStompClient() 호출
[lib/socket/createMeetingStompClient] ← STOMP 클라이언트 팩토리
    ↓ 메시지 수신 시
[lib/socket/meetingRealtimeCache] ← TanStack Query 캐시 직접 조작
```

## 2. WebSocket 연결 수립 (Lifecycle)

연결은 컴포넌트가 마운트될 때 시작되고, 언마운트될 때 정리됩니다. (`useEffect` cleanup)

```
[컴포넌트 마운트]
    ↓
1. createMeetingStompClient({ onConnect, onStompError, ... }) 생성
2. client.activate() → WebSocket 연결 시도
    ↓ (연결 성공 시) onConnect() 콜백 실행
3. setIsConnected(true)
4. resubscribe() → 발제(Presentation) + 채팅(Chat) + 에러 채널 구독
5. invalidateRealtimeQueries() → 연결 직후 최신 데이터로 동기화
    ↓
[컴포넌트 언마운트]
1. subscriptions.presentation?.unsubscribe()
2. subscriptions.chat?.unsubscribe()
3. subscriptions.error?.unsubscribe()
4. client.deactivate()
```

## 3. 구독 채널 및 권한 제어

채널을 구독하기 전, **사용자의 권한을 먼저 확인**합니다. 이를 통해 권한 없는 팀의 채팅을 수신하는 보안 문제를 방지합니다.

| 채널 | 구독 조건 |
| :--- | :--- |
| **발제 채널** | 운영진(`isStaff`) 이거나 본인 팀(`presentationTeamId === myTeamId`)인 경우에만 구독 |
| **채팅 채널** | 채팅이 열려있고(`isChatOpen`), 운영진이거나 본인 팀의 채팅인 경우에만 구독 |
| **에러 채널** | 항상 구독 (사용자 개인 에러 수신용) |

## 4. `latestRef` 패턴 (Stale Closure 방지)

WebSocket 콜백 함수는 최초 등록 시의 값을 클로저로 캡처합니다. `clubId`, `myTeamId` 등이 변경되어도 콜백이 오래된 값을 참조하는 **Stale Closure 버그**가 발생할 수 있습니다.

이를 방지하기 위해 `latestRef`라는 `useRef` 패턴을 사용합니다.

```typescript
const latestRef = useRef({ clubId, myTeamId, ... });

// props가 변경될 때마다 ref를 최신 값으로 업데이트
useEffect(() => {
  latestRef.current = { clubId, myTeamId, ... };
}, [clubId, myTeamId, ...]);

// 콜백에서는 항상 ref를 통해 접근 (항상 최신값 보장)
const callback = () => {
  const current = latestRef.current; // ✅ 항상 최신 값
};
```

## 5. 메시지 수신 후 캐시 업데이트 방식

메시지를 수신했을 때 화면을 즉시 업데이트하는 방법에는 2가지가 있습니다.

*   **캐시 직접 조작 (`setQueryData`)**: `appendChatMessageToChatsCache`는 서버에 재요청 없이 TanStack Query 캐시에 메시지를 직접 추가합니다. 채팅처럼 즉각적인 반영이 필요할 때 사용합니다.
*   **캐시 무효화 (`invalidateQueries`)**: `applyPresentationEventToTopicsCache`는 발제 이벤트 수신 시 관련 쿼리를 무효화하여 서버와 재동기화합니다.

## 6. 메시지 발행 (Publish)

훅은 채팅 전송과 발제 선택을 위한 두 가지 발행 함수를 노출합니다.

*   **`publishChatMessage(teamId, content)`**: 발행 전 연결 여부, 빈 메시지 여부, 400자 초과 여부, 팀 권한을 검증합니다. 검증 실패 시 `Error`를 throw 합니다.
*   **`publishPresentation(teamId, topicId, isSelected)`**: 발제문의 선택/취소 이벤트를 서버로 전송합니다.
