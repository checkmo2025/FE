# 01. 프로젝트 개요 및 구조

이 문서는 CheckMo 프로젝트의 전반적인 기술 스택과 코드 구조를 설명하여, 새로운 엔지니어가 프로젝트의 구성을 빠르게 파악하는 것을 돕습니다.

## 1. 기술 스택 (Tech Stack)

CheckMo 프론트엔드는 최신 웹 표준과 생산성을 고려한 기술들로 구성되어 있습니다.

| 분류 | 기술 | 비고 |
| :--- | :--- | :--- |
| **Core** | Next.js 16 (App Router), React 19, TypeScript | 최신 기능을 활용한 SSR/CSR 병행 |
| **State** | Zustand | 가볍고 직관적인 전역 상태 관리 |
| **Data Fetching** | TanStack Query (v5) | 서버 상태 관리 및 캐싱 |
| **Styling** | Tailwind CSS 4 | 유틸리티 퍼스트 스타일링 |
| **Animation** | Framer Motion | 부드러운 UI 인터랙션 및 전환 |
| **Form/Validation** | Zod | 데이터 스키마 검증 및 타입 추론 |
| **Interaction** | @dnd-kit | 드래그 앤 드롭 구현 |
| **Real-time** | @stomp/stompjs | WebSocket 통신 (독서 모임 채팅 등) |

## 2. 디렉토리 구조 (Project Structure)

`src` 디렉토리를 중심으로 관심사 분리(SoC) 원칙에 따라 구조화되어 있습니다.

```text
src/
├── app/            # Next.js App Router (Route Groups: (main), (public), (admin))
├── components/     # 재사용 가능한 UI 컴포넌트
│   ├── auth/       # 인증 관련 컴포넌트 (AuthProvider 등)
│   ├── base-ui/    # 공통 UI 요소 (Button, Input 등)
│   ├── layout/     # 페이지 레이아웃 (Header, BottomNav 등)
│   └── ...         # 도메인별 컴포넌트
├── services/       # API 호출 로직 (authService, clubService 등)
│   └── mappers/    # [추후 도입] DTO <-> Model 변환 로직
├── hooks/          # 커스텀 훅 (queries, interactions 등)
├── store/          # Zustand 스토어 (useAuthStore 등)
├── lib/            # 라이브러리 설정 (api/client 등)
├── types/          # 전역 TypeScript 타입 정의
├── utils/          # 유틸리티 함수
├── constants/      # 상수 정의 (API 엔드포인트 등)
└── data/           # 정적 데이터 또는 Mock 데이터
```

### 주요 경로 별칭 (Path Alias)
`tsconfig.json`에 정의된 `@/*` 별칭을 사용하여 `src/` 디렉토리에 접근합니다.
- 예: `import { apiClient } from "@/lib/api/client";`

## 3. 라우팅 전략 (Route Groups)

Next.js의 Route Groups 기능을 사용하여 권한 및 레이아웃에 따라 페이지를 분류합니다.

*   **(public)**: 비로그인 사용자도 접근 가능한 페이지 (회원가입, 계정 찾기 등)
*   **(main)**: 서비스의 주요 기능을 담당하는 페이지 (홈, 모임 리스트, 설정 등)
*   **(admin)**: 관리자 전용 페이지 (`/admin` 하위 경로)

## 4. 개발 환경 설정 및 컨벤션

### 실행 방법
```bash
pnpm install    # 의존성 설치
pnpm dev        # 로컬 개발 서버 실행
```

### 브랜치 및 커밋 전략 (README 기준)
*   **Main**: 배포용 브랜치
*   **Develop**: 기능 통합 브랜치
*   **Naming**: `feat-[이슈번호]`, `bug-[이슈번호]` 등
*   **Commit**: `feat:`, `refactor:`, `bug:`, `style:`, `docs:` 등의 접두사 사용

### 코드 컨벤션
*   **컴포넌트/파일명**: `PascalCase`
*   **변수/함수명**: `camelCase`
*   **상수**: `UPPER_SNAKE_CASE`
*   **CSS**: Tailwind CSS 4를 기본으로 하며, 필요한 경우 CSS Variables 활용
