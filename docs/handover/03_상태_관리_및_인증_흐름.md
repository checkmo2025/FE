# 03. 상태 관리 및 인증 흐름

CheckMo 프론트엔드는 전역 상태의 무분별한 사용을 피하고(KISS 원칙), 최소한의 클라이언트 상태만을 중앙에서 관리합니다. 가장 대표적인 전역 상태는 **사용자 인증 정보**와 **전역 UI(모달)** 입니다.

## 1. 전역 상태 관리 (Zustand)

프로젝트의 클라이언트 전역 상태 관리는 `Zustand`를 사용합니다.
단순하고 직관적인 API를 제공하여 보일러플레이트를 최소화합니다.

### 주요 스토어: `useAuthStore`
`src/store/useAuthStore.ts`는 사용자 인증 상태와 관련된 모든 전역 상태를 관장합니다.

*   **상태 (State)**
    *   `user`: 로그인된 사용자 정보 (null이면 비로그인)
    *   `isLoggedIn`: 로그인 여부 (boolean)
    *   `isLoginModalOpen`: 전역 로그인 모달 렌더링 여부
    *   `isInitialized`: 앱 로드 시 세션 확인(초기화) 완료 여부
*   **액션 (Actions)**
    *   `login(user)`: 로그인 성공 시 사용자 정보 저장 및 상태 업데이트
    *   `logout()`: 로그아웃 처리 및 `js-cookie`를 사용해 토큰 쿠키 제거
    *   `openLoginModal()` / `closeLoginModal()`: 전역 로그인 모달 제어

## 2. 인증 및 세션 초기화 흐름 (Auth Flow)

페이지를 새로고침하거나 앱에 처음 진입할 때, 사용자의 로그인 상태를 매번 복구해야 합니다. 이를 위해 `AuthProvider` 컴포넌트를 사용합니다.

### AuthProvider (`src/components/auth/AuthProvider.tsx`)
1.  **초기화 (Initialization)**: 컴포넌트 마운트 시 `authService.getProfile()` API를 호출하여 서버에 세션을 확인합니다.
2.  **성공 (Success)**: 응답이 성공적이면 `useAuthStore`의 `login` 액션을 호출하여 전역 상태를 업데이트합니다.
3.  **실패 (Failure)**: 세션이 만료되었거나 비로그인 상태인 경우 (401 에러 등), `logout` 액션을 호출하여 상태를 초기화합니다.
4.  **완료 (Completion)**: 성공/실패 여부에 관계없이 `setInitialized(true)`를 호출하여 앱 로딩이 완료되었음을 알립니다.

*이 흐름 덕분에 모든 하위 컴포넌트는 `useAuthStore`의 `user` 상태만 구독하면 항상 최신의 로그인 상태를 신뢰하고 사용할 수 있습니다.*

## 3. 전역 로그인 모달 (Global Login Modal)

특정 액션(예: 좋아요 누르기, 모임 가입하기)을 시도할 때 로그인이 필요한 경우, 페이지를 이동시키는 대신 **전역 로그인 모달**을 띄웁니다.

1.  **호출**: 어떤 컴포넌트에서든 `useAuthStore().openLoginModal()`을 호출합니다.
2.  **렌더링**: 최상위 레이아웃(`src/app/layout.tsx`)에 배치된 `<GlobalLoginModal />`이 상태 변화를 감지하고 모달을 띄웁니다.
3.  **로그인 수행**: 모달 내에서 로그인을 완료하면 상태가 업데이트되고 모달이 닫힙니다. 사용자는 기존에 보던 페이지에서 이탈하지 않고 작업을 계속할 수 있습니다.
