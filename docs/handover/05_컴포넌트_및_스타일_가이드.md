# 05. 컴포넌트 및 스타일 가이드

이 문서는 CheckMo 프로젝트의 일관된 UI 구현을 위한 컴포넌트 설계 및 스타일링 가이드라인입니다.

## 1. 스타일링 기술 스택

### Tailwind CSS v4
프로젝트의 기본 스타일링 도구로 **Tailwind CSS v4**를 사용합니다.
*   **유틸리티 퍼스트**: CSS 파일을 별도로 생성하지 않고 컴포넌트 내부에서 클래스 조합만으로 디자인을 완성합니다.
*   **일관성**: `globals.css` 및 `tailwind.config.js` 등에 정의된 테마 컬러(예: `bg-[#7B6154]`)와 간격 시스템을 준수하여 통일감을 유지합니다.

### Framer Motion
부드러운 화면 전환, 모달 팝업, 요소 드래그 등 동적인 인터랙션에는 **Framer Motion** 라이브러리를 적극 활용합니다.
*   `<motion.div>` 컴포넌트를 사용하여 `initial`, `animate`, `exit` 속성으로 선언적인 애니메이션을 구현합니다.

## 2. 컴포넌트 구조화 (SoC 원칙)

컴포넌트의 재사용성을 높이기 위해 `src/components/` 디렉토리를 역할에 따라 엄격히 분리합니다.

### Base UI (`src/components/base-ui/`)
가장 하위 레벨의 순수 UI 컴포넌트들입니다. (버튼, 인풋, 모달 컨테이너 등)
*   **특징**: 비즈니스 로직(API 페칭, 복잡한 상태 관리 등)을 절대로 포함하지 않습니다. (Dumb Components)
*   **Props 의존**: 모든 데이터와 콜백 함수는 부모로부터 Props로 전달받습니다.

### Domain / Feature 컴포넌트
특정 기능이나 도메인(예: 독서 모임, 프로필, 스토리 등)에 종속된 컴포넌트들입니다.
*   **특징**: 필요한 경우 내부에서 Custom Hook을 호출하여 데이터(Server State)를 불러오거나 이벤트 액션을 처리할 수 있습니다. (Smart Components)
*   **조합**: 내부 UI 렌더링 시 가급적 Base UI 컴포넌트들을 조립하여 화면을 구성합니다.

### Layout (`src/components/layout/`)
페이지 전반에 걸쳐 공통으로 사용되는 레이아웃 껍데기 컴포넌트들입니다. (Header, BottomNav 등)

## 3. 반응형 웹 디자인 (Responsive Design)

CheckMo는 모바일, 태블릿, 데스크탑을 모두 지원하는 반응형 웹으로 구축됩니다.
Tailwind의 미디어 쿼리 접두사를 활용하여 뷰포트 크기에 대응합니다. (설정된 커스텀 브레이크포인트 준수 요망)

*   **Mobile First**: 기본 클래스는 모바일 화면을 기준으로 작성합니다.
*   **Tablet (`t:`)**: 태블릿 사이즈 이상일 때 적용될 스타일 (예: `t:pb-0`, `t:gap-6`)
*   **Desktop (`d:`)**: 데스크탑 사이즈 이상일 때 적용될 스타일 (예: `d:flex-row`, `d:max-w-[332px]`)

## 4. 컴포넌트 작성 컨벤션 (KISS & DRY)

*   **조건부 렌더링**: 로딩 스피너, 에러 상태 등은 `isLoading`, `isError` 등의 Boolean Props를 받아 명시적으로 렌더링합니다.
*   **컴포넌트 분할**: 한 파일(컴포넌트)의 라인 수가 지나치게 길어지면, 논리적인 UI 블록 단위로 분할하여 다른 파일로 추출하는 것을 권장합니다.
